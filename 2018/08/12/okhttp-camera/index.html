使用OKHttp

首先gradle文件中添加OkHttp库的依赖

步骤：
1.创建客户端（client），即OkHttpClient的实例化对象
-
OnHttpClient client = new OkHttpClient();
-

2.创建要求（request），即Request对象
-
Request request = new Request.Builder().url(“http://baidu.com").build();
-

3.创建发送（call），发送请求并获取服务器的放回数据，即创建Call对象然后调用execute（）方法返回数据
-
Response response = client.newCall(request).excute();
-其中Response对象就是我们的返回数据了

4.将数据转换成具体内容
-
String responseData = response.body().string();
-

上面的知识发起一条GET请求，而发起POST请求会麻烦一些。
要多构造出一个RequestBody的对象存放待提交的参数，

然后再上面的创建请求的步骤中加入此RequestBody对象

Request request = new Request.Builder()
    .url("http://baidu.com")
    .post(requestBody)
    .build();

调用拍照功能

逆向思维：

1.不再是用setImageResourceId方法设置Id，而是用setImageBitmap设置bitmap

|
| 要通过设置Bitmap来获得图片，就要得到Bitmap对象
|
v
2.通过Bitmap工厂解析流，而输入流通过openInputStream（Uri对象）方法将Uri对象转换成InputStream对象

|
| 这样看来取得图片的Uri就可以获得图片，那么最主要是获得imageUri
|
v
3.通过FileProvider的getUriforFile(1, 2, 3)方法传入三个参数：
第一个：Context对象。
第二个：任意唯一的字符串。
第三个：File对象。（缓存拍照图片的文件）

上述为android 7.0 之后的取得Uri对象方式，在开发中还要加判断语句 Build.VERSION.SDK_INT >= 24 ,
若是版本较低的，用：imageUri = Uri.fromFile(outputImage)
|
| 要将获得存储拍照后图片文件的Uri，就要创建对应的文件File对象
|
v
4.通过 new File（父路径：getExternalCacheDir() ，文件名：”output_image.jpg”）
和createNewFile()来创建文件。

其中getExternalCacheDir()是一个获取应用关联缓存目录，即new一个File所需要的父路径。
